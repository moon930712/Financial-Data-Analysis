# 포트폴리오 관리 가이드라인 (AI Assistant용)

## 1. 목적 (Purpose)
- 이 레포지토리는 사용자의 **이직 및 구직을 위한 포트폴리오(Portfolio)** 용도입니다.
- 채용 담당자가 코드, 커밋 히스토리, 이슈 관리 능력을 평가할 수 있도록 **깔끔하고 프로페셔널한 상태**를 유지해야 합니다.

## 2. 보안 및 기밀 유지 (Security & Public Policy)
- **절대 커밋 금지**: 실제 회사의 사내 비밀 데이터베이스 접속 정보, 민감한 개인정보, 외부 유출이 금지된 실데이터(row data).
- **공공성 준수**: 깃허브에 올리는 모든 내용(코드, 문서, 스키마 구조 등)은 외부에 공개되어도 법적, 윤리적 문제가 없는 형태로 비식별화/추상화하거나 공공 데이터 수준으로 가공해야 합니다. (`.env`, `db.txt` 등 서버 접속 정보는 반드시 `.gitignore`로 제외)

## 3. 커밋 및 업로드 컨벤션 (Commit Convention)
- 채용 담당자가 작업의 흐름을 쉽게 파악할 수 있도록 **의미 있는 단위로 커밋**해야 합니다.
- **커밋 메시지 형식**: Conventional Commits 형식을 지향합니다.
  - `feat: 데이터베이스 스키마 추출 자동화 스크립트 작성`
  - `docs: 포트폴리오 및 프로젝트 README 작성`
  - `refactor: 보안 강화를 위해 하드코딩된 DB 정보 분리`
- 커밋 본문(body)에는 간단하게 '왜 해당 작업을 했는지(Why)'를 명시하여 문제 해결 능력을 어필합니다.

## 4. 코드 퀄리티 (Code Quality)
- 변수명, 함수명은 직관적이고 일관성 있게 작성합니다.
- 주석(Comments)은 한글로 친절하게 달되, 왜 이런 로직을 구현했는지 그 '의도'를 덧붙이는 것이 좋습니다.
- 잠재적 에러 상황을 우아하게 처리(Exception Handling)하여 프로페셔널한 인상을 남깁니다.
