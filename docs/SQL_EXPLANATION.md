# 순수 SQL 기반 다중 요인 Z-Score 랭킹 모델 쿼리 기술서

이 문서는 `src/industry_zscore_ranking.sql` 파일 내부에 작성된 **PostgreSQL 전용 윈도우 함수(Window Function) 및 CTE(Common Table Expression) 쿼리**의 파이프라인 구조를 설명합니다.

파이썬(Pandas)과 같은 외부 애플리케이션의 메모리 가공 처리 없이, **데이터베이스 엔진(DBMS)의 자체 연산만을 사용하여 시계열 통계와 표준화(Z-Score)를 한 번에 처리**하기 위해 설계되었습니다.

---

## 🏗️ 쿼리 블록별 상세 아키텍처

쿼리는 가독성과 디버깅 효율을 높이기 위해 총 8개의 **CTE (WITH 문)**와 1개의 **최종 SELECT 문**으로 연결되어 있습니다.

### 1️⃣ 최신 영업일 조달 (`latest_date`)
데이터 셋(`krx_stocks_fundamental_info`)에 존재하는 가장 최신 날짜(`MAX(date)`)를 스칼라로 추출하여, 이후 과거 3년 데이터를 가져올 때의 기준점(Anchor)으로 사용합니다.

### 2️⃣ 시계열 평균 산출 (`historical_stats`)
최신 영업일 기준 `- INTERVAL '3 years'` (과거 3년) 조건으로 테이블을 필터링한 뒤, 각 종목(코드)별 과거 3년간의 평균 PER, PBR을 계산합니다. `AVG()` 적용 시 0인 값을 제외하기 위해 `NULLIF`를 사용하여 데이터 집계의 왜곡을 막았습니다.

### 3️⃣ 일일 펀더멘탈 (`current_stats`)
과거 데이터와 비교할 오늘(최신 날짜) 기준의 현행 펀더멘탈(PER, PBR)만 개별적으로 추출합니다.

### 4️⃣ 업종 분류망 추출 (`market_info`)
`visual.vsl_anly_stocks_price_subindex01` 테이블에서 가장 최근 일자의 데이터를 조회하여, 종목별 공식 WICS(업종/섹터) 분류명과 종목명을 추출해냅니다.

### 5️⃣ 종목 단위 테이블 결합 (`combined_stock_data`)
1번부터 4번까지 추출한 CTE들을 조인하여 개별 종목 관점의 평가 테이블 뷰(View)를 생성합니다.
- **ROE 근사 도출**: `PBR = PER * ROE` 공식을 역산(`PBR / PER`)하여 수익성을 측정. (순이익 적자 시 패널티 부여)
- **가치(Value) 비율 도출**: 현재 PBR을 3년 평균 PBR로 나누어 할인율 측정.
- **저평가 플래그(`is_undervalued`)**: 과거 평균 기준 대비 90% 이하로 하락한 종목에만 `1.0` 마킹 부여.

### 6️⃣ 업종 단위 팩터 집계 (`industry_aggregation`)
개별 종목 레벨에서 평가된 모든 지표들을 **`GROUP BY industry`**를 통해 '업종' 레벨로 묶습니다.
- 여기서 업종별 소속 종목의 기초통계 평균(`AVG`)을 도출하며, 소수 테마성 왜곡을 막기 위해 5개 종목 이상인 섹터만 남깁니다(`HAVING COUNT >= 5`).

### 7️⃣ 저평가 밀집도 연산 (`industry_metrics`)
업종 내의 전체 종목 수 대비 저평가 마킹된 종목들의 수를 백분율(%)로 계산하여, 해당 업종이 얼마나 동반 하락했는지 확인하는 '바닥론 퀄리티(Density)'를 산출합니다.

### 8️⃣ 순수 SQL Z-Score 표준화 (`zscore_calculation`)
스케일이 서로 다른 집계 지표(수익성, 가치, 밀집도)들을 하나의 점수표에서 평가하기 위해 `Window Function` 지표인 **`STDDEV_POP`**(모표준편차)와 **`AVG`**(평균)를 함께 사용하여 표준점수(Z-score) 수식 $Z = (X - \mu) / \sigma$ 을 SQL 내에서 강제로 구현합니다.

### 9️⃣ 최종 정렬 (`SELECT`)
도출된 3가지 Z-Score 값에 자산 배분 비중(가중치: `밀집도 0.4`, `수치적 저평가 0.3`, `기초 펀더멘탈 0.3`)을 곱하여 합산한 결과 값을 리턴합니다.
결과 테이블 맨 앞에 **`ROW_NUMBER() OVER(ORDER BY ... DESC)`** 함수를 박아 넣어 1위부터 순서대로 번호를 매깁니다.

---
**✅ 결론**: 전체 DB 데이터를 Pandas로 불러오는 병목(Bottleneck) 현상을 원천 차단하고 RDBMS의 집계 기능을 100% 한계까지 활용한 하드코어 쿼리 분석 아키텍처입니다.
