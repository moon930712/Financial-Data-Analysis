# 프로젝트 관리 및 기여 가이드라인 (Project Guidelines)

## 1. 목적 (Purpose)
- 이 레포지토리는 금융 데이터 분석 및 퀀트 스크리닝 파이프라인 고도화를 위한 **독립적인 오픈 리서치 공간**입니다.
- 프로젝트 참여자나 리뷰어가 코드, 커밋 히스토리, 이슈 관리 패턴을 통해 아키텍처의 흐름을 쉽게 파악할 수 있도록 **깔끔하고 프로페셔널한 상태**를 유지합니다.

## 2. 보안 및 기밀 유지 (Security & Public Policy)
- **절대 커밋 금지**: 사내 비밀 데이터베이스 접속 정보, 민감한 개인정보, 외부 유출이 금지된 실데이터(raw data).
- **무결성 준수**: 깃허브에 공유되는 모든 쿼리와 아키텍처 코드는 외부에 공개되어도 법적, 윤리적 문제가 없는 형태로 설계해야 합니다. (`.env`, `db.txt` 등 접속 정보는 무조건 `.gitignore`로 제외)

## 3. 커밋 컨벤션 (Commit Convention)
- 협업 및 히스토리 열람을 고려하여 **의미 있는 단위로 커밋**합니다.
- **커밋 메시지 형식**: Conventional Commits 형식을 지향.
  - `feat: 데이터베이스 스키마 추출 자동화 스크립트 작성`
  - `docs: 프로젝트 README 및 퀀트 전략 구조 문서 업데이트`
  - `refactor: 보안 강화를 위해 하드코딩된 DB 정보 분리`
- 커밋 본문에는 간단하게 작업의 배경(Why)과 핵심 개선사항을 명시합니다.

## 4. 코드 퀄리티 (Code Quality)
- 변수명, 함수명은 직관적이고 일관성 있게 작성합니다.
- 주석(Comments)은 한글로 친절하게 달되, 왜 이런 로직을 구현했는지 그 '의도'를 덧붙이는 것이 좋습니다.
- 잠재적 에러 상황을 우아하게 처리(Exception Handling)하여 프로페셔널한 인상을 남깁니다.
