# 순수 SQL 기반 업종별 Z-Score 랭킹 모델 (심플 모멘텀 버전)

이 문서는 `src/industry_zscore_ranking.sql` 파일에 구현된 **업종 중심 다중 요인 Z-Score 랭킹 모델**의 아키텍처를 설명합니다.

본 모델은 복잡한 개별 종목 지표 대신, **업종 전체의 시계열 데이터**와 **최근 1개월 수급 추이**를 결합하여 가장 활력이 넘치고 저평가된 업종을 포착하는 데 목적이 있습니다.

---

## 🏗️ 쿼리 블록별 상세 아키텍처 (CTE 구성)

쿼리는 논리적 흐름에 따라 총 8개의 블록으로 구성되어 있습니다.

### 1️⃣ 기준 날짜 추출 (`latest_date`)
데이터 셋에 존재하는 가장 최신 영업일을 추출하여 모든 분석의 기준점으로 설정합니다.

### 2️⃣ 업종 및 데이터 정제 (`industry_mapping`)
- **시장 구분**: KOSPI/KOSDAQ 정보를 통합합니다.
- **데이터 클렌징**: 원본 데이터의 ROE 필드에서 발생할 수 있는 공백이나 문자열 오류를 제거하고 수치형(Numeric)으로 안전하게 변환합니다. (기존에 발생하던 `invalid input syntax` 오류를 방지하기 위해 `trim` 및 `NULLIF` 로직 적용)

### 3️⃣ 업종별 3년 평균 집계 (`industry_historical_stats`)
각 업종의 **과거 3년 평균 PBR, ROE**를 산출합니다. 이는 업종 간의 절대적 차이를 극복하고, "해당 업종이 평소 실력 대비 얼마나 좋아졌는가"를 측정하는 기준이 됩니다.

### 4️⃣ 업종별 현재 상태 집계 (`industry_current_stats`)
최신 영업일 기준의 업종 평균 PBR과 소속 종목 수를 집계합니다. (데이터 신뢰성을 위해 종목 수 5개 미만 업종은 제외)

### 5️⃣ 거래량 수급 분석 (`industry_volume_momentum`)
- **최근 1개월(약 20영업일) 평균 거래량**과 **이전 3개월 평균 거래량**을 비교합니다.
- 이를 통해 수급이 평소보다 강력하게 유입되고 있는 '수급 유망 업종'을 포착합니다.

### 6️⃣ 핵심 지표 산출 (`industry_metrics`)
- **[가치] PBR 비율**: 현재 PBR / 3년 평균 PBR (과거 대비 저평가 수준)
- **[수급] 거래량 증가율**: 최근 1개월 / 이전 3개월 (수급 모멘텀 수준)
- **[수익] ROE**: 해당 업종의 현재 수익성 수준

### 7️⃣ 업종 간 표준점수화 (`zscore_calculation`)
서로 다른 단위(배수, 증가율 등)를 가진 지표들을 공정하게 비교하기 위해 SQL 윈도우 함수(`STDDEV_POP`)를 활용하여 **Z-Score(표준점수)**를 산출합니다.

### 8️⃣ 최종 가중치 랭킹 (`SELECT`)
최종 점수는 다음과 같은 가중치로 합산되어 상위 업종을 선정합니다.
- **수급(거래량 증가) 40%**
- **가치(PBR 할인) 30%**
- **수익성(ROE) 30%**

---

## ✅ 모델의 강점
1. **자기 비교 방식**: 업종별 고유 특성에 따른 왜곡을 최소화하여 성장주와 가치주를 공정하게 비교합니다.
2. **수급 모멘텀 반영**: 단순 저평가뿐만 아니라 시장의 수급이 실제로 들어오고 있는지를 확인합니다.
3. **데이터 안정성**: 문자열 데이터 처리를 강화하여 쿼리 실행 시의 타입 오류를 원천 차단했습니다.
