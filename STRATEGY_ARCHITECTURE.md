# 퀀트 모델 설계 노트 (Z-Score 매트릭스)

이 문서는 `quant_industry_zscore.py` 작성 시 설계한 쿼리 구조와 수식 논리를 기록한 노트입니다.

## 1. 데이터 추출 (SQL)
Python 메모리로 전체를 불러오기 전, DB단(PostgreSQL)에서 시계열 집계를 미리 처리하여 속도를 최적화했습니다.

- latest_date: 가장 최근 영업일 동적 추출
- historical_stats: 과거 3년치 데이터를 바탕으로 종목별 평균 PBR, PER 산출
- current_stats: 최근 영업일 단일 펀더멘탈 추출
- 조인: 업종 분류(WICS)를 기준으로 모든 시계열 데이터를 Python으로 가져옵니다.

## 2. 지표 가공 (Python/Pandas)
단순한 절대평가보다는 업종 내 상대평가와 시계열 분석을 결합하는 데 초점을 맞췄습니다.

- ROE 근사: 순이익 데이터가 불규칙할 경우를 대비하여 PBR/PER 값을 통해 자본이익률을 역산했습니다.
- 저평가 마킹: 개별 종목의 현재 PBR이 과거 3년 평균 대비 90% 이하일 때 마킹 플래그를 추가했습니다.

## 3. Z-Score 표준화 및 결합
업종(Industry)별로 그룹화한 뒤, 스케일이 다른 팩터들을 표준화(Z)하여 결합했습니다.

- 산출 팩터:
  - Density: 업종별 저평가 종목의 발생 비율 (바닥론)
  - Value: 업종의 역사적 평균 PBR 대비 현재 할인율
  - Fundamental: 업종 평균 추정 ROE (가치함정 회피)
  
- 최종 산출: 각 Z-Score를 합산해 점수가 높은 업종부터 순위를 배열했습니다. 가치 지표가 작을수록 유리한 필드는 역수를 취하여 일관된 방향성을 갖추도록 맞추었습니다.
